  - **理由**: 拡張性と保守性の向上、責務の明確な分離

### データベース選択
- **初期案**: SQLite（開発の簡易さ）
  - **現在**: PostgreSQL
  - **理由**: 本番環境での堅牢性、JSON型のサポート、トランザクション機能

### API設計
- **初期案**: RPC風のエンドポイント
  - **現在**: RESTful API
  - **理由**: 標準的なアプローチ、クライアント連携の容易さ、拡張性
